Option 1: Research a new web technology

Overview

Each member should take turns presenting the technology they learned about, what it is designed for, how it works, and it's strengths and limitations, etc.

At the end of this activity, each team member should write a short summary of their technologies presented (including their own and those of the other team members), and submit it to I-Learn.

Core Requirements

  1. Present your new technology to the team.

  2. Actively participate in learning from at least one other team member.

  3. Learn from ALL team members on the team.

Option 2: Create a livechat room

Overview

The primary task here is to combine some of your learning from the past semester and create a simple yet very useful web application. While the application does not have a backend, one could be easily implemented.

Start working with the files located within the zip folder here.

Core Requirements

  1. When a user logs in using the ta12-login.ejs, their username is stored as a session variable. There cannot be duplicate usernames. If a user enters a duplicate username, they are notified.

  2. When a user logs in, other users logged into the chat are notified via an "admin" user.

  3. When a user sends a message, it is appended to their own chatbox and all other connected users. (basically, the page follows the basic operations of a messaging application)

This application has a lot of potential for creativity. For example, you could:

  • Include the time a message was sent in the message.

  • Make the messages persistent using MongoDB.

  • Notify users when a client disconnects.

  • Use socket.io rooms (instead of a global chat)
